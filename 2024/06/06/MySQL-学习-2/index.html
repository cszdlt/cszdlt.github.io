MySql 数据库组件

分为连接管理器、查询管理器以及数据处理器。

连接管理器

连接管理器也叫客户端管理器，主要用于管理客户端的连接以及鉴权判断，接收、响应请求等。

查询管理器

查询管理器又分为查询分析、查询预处理、查询优化和查询执行等几个模块。

  • 查询分析：通过词法法分析和语法分析检查该查询是否合法。
  • 查询预处理：检查查询语句所需的表是否存在、字段是否存在、运算是否合法。
  • 查询优化：根据表信息以及查询语句，优化查询的执行步骤。
    • 对于存在子查询的语句，优化掉子查询。
    • 对于存在连接的语句，优化连接效率。
    • 处理索引覆盖等。
  • 查询执行：跟数据引擎交互的模块，跟数据引擎交互，并判断是否完成等。
    • 对于复合索引，遵从左匹配原则，如果索引为 (a,b)，查询条件为 b 是无法利用上该索引的。
    • 对于复合索引，如果右边的索引失效了，早期的 MySQL 在从 数据引擎 中拿到数据时，执行器会再去主索引获取该条记录的信息，再判断索引条件是否匹配；新版的 MySQL 在这种情况下，会在执行器中判断拿到的数据是否符合索引条件，符合之后再去主索引树中获取具体信息。(索引下推)
    • 执行器每拿到一条数据就将一条数据返回给客户端了。
